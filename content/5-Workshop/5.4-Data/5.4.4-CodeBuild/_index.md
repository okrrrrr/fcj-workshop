### Creating AWS CodePipeline

After configuring AWS CodeBuild, the next step is to build a **Continuous Integration and Continuous Deployment (CI/CD)** pipeline using the **AWS CodePipeline** service.

Access the **AWS CodePipeline** service, select **Create pipeline**, and choose the **Build custom pipeline** option to manually configure the entire deployment process.

![](/images/5-Workshop/5.4-Data/image-15.png)

### Configuring the Pipeline

Next, enter the Pipeline name and configure basic details such as the **Execution mode** and the **IAM Service Role** so that AWS CodePipeline has the necessary permissions to access required services during deployment.

![](/images/5-Workshop/5.4-Data/image-16.png)

Once completed, the Pipeline will use this IAM Role to orchestrate the entire process, from fetching source code and building to deploying the application.

### Connecting to GitHub

In the **Source Stage** step, select **GitHub (via GitHub App)** as the source provider for the Pipeline.

Then, connect your GitHub account and select the repository and branch to monitor. Whenever changes occur on the configured branch, AWS CodePipeline will automatically trigger the build and deployment processes.

![](/images/5-Workshop/5.4-Data/image-17.png)

Review the GitHub connection details—including the repository, branch, and output artifact format—before proceeding to the next step.

![](/images/5-Workshop/5.4-Data/image-18.png)

Connecting to GitHub enables the automation of the entire CI/CD process. Every time a developer performs a **git push**, the Pipeline automatically fetches the latest version of the source code to proceed with the build and deployment.

### Configuring the Build Stage

After completing the Source Stage, configure the **Build Stage** by selecting the project previously created in AWS CodeBuild.

The pipeline will use this project to execute the steps defined in the **buildspec.yml** file, which include setting up the environment, building the application, and generating build artifacts.

![](/images/5-Workshop/5.4-Data/image-19.png)

### Configuring the Deploy Stage

The final step is to configure the **Deploy Stage**.

Select **Amazon S3** as the deployment service, then specify the destination bucket and the build artifact generated by AWS CodeBuild. Additionally, enable the **Extract file before deploy** option to automatically extract files prior to deployment.

![](/images/5-Workshop/5.4-Data/image-20.png)

Once configuration is complete, AWS CodePipeline will automatically execute the following process:

- Retrieve the latest source code from GitHub.
- Trigger AWS CodeBuild to compile and package the application.
- Generate the build artifact.
- Deploy the build artifact to Amazon S3.
- Monitor the status of each stage within the pipeline.

### Completion

Upon completing the steps above, the system has successfully established a **CI/CD** pipeline using AWS CodePipeline. From this point forward, whenever the source code on GitHub is updated, the pipeline will automatically fetch the new code, perform the build via AWS CodeBuild, and deploy the application to Amazon S3 without requiring manual intervention. This streamlines deployment, minimizes errors, and enhances version management efficiency throughout the system development process.
